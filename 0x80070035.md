1. Buka **Registry Editor di Windows 10** pada path:

```
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\LanmanWorkstation\Parameters
```

Di sana ada key **AllowInsecureGuestAuth** dengan nilai **1**.
Artinya **akses guest sharing sudah diaktifkan**, biasanya ini dilakukan agar komputer bisa membuka sharing tanpa login (sering dipakai di jaringan kantor kecil atau NAS lama).

Jadi dari sisi **registry sebenarnya sudah benar** 👍

Namun error **0x80070035** biasanya masih bisa muncul jika ada beberapa hal lain yang belum aktif.

Saya sarankan cek 3 hal ini lagi (paling sering jadi penyebab).

2. Buka edit group policy editor
Computer Configuration> Administrative Templates> Network> Lanman Workstation
Enable: Enable insecure guest logons

---

# 1. Aktifkan SMB Client

Kadang registry sudah benar tapi **SMB belum aktif**.

Tekan **Windows + R**

```
optionalfeatures
```

Cari dan centang:

```
SMB 1.0/CIFS File Sharing Support
```

Minimal centang:

```
SMB 1.0/CIFS Client
```

Lalu **restart komputer**.

---

# 2. Pastikan Service Sharing Aktif

Tekan **Windows + R**

```
services.msc
```

Pastikan service ini **Running**:

* Function Discovery Provider Host
* Function Discovery Resource Publication
* Server
* Workstation

Kalau **Stopped → klik Start**.

---

# 3. Coba Akses Pakai IP

Kadang nama komputer tidak terbaca di network.

Di **File Explorer** ketik:

```
\\192.168.1.xxx
```

contoh:

```
\\192.168.1.10
```

Jangan pakai:

```
\\NAMA-PC
```

---

# 4. Jika Masih Error (Sering Terjadi di Kantor)

Reset network stack.

Buka **Command Prompt (Run as Administrator)**

```
netsh winsock reset
netsh int ip reset
ipconfig /flushdns
```

Restart komputer.

---

Biasanya masalah tinggal di:

* permission folder
* firewall
* akses pakai hostname
